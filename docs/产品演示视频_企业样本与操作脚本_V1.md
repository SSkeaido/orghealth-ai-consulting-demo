# 企业健康初诊 V3.1：产品演示视频样本与操作脚本

在线地址：<https://sskeaido.github.io/orghealth-ai-consulting-demo/?v=3.1>

## 使用原则

- 以下企业、联系人、经营数据均为虚构演示信息。
- 问题描述包含“现象、影响、时间、已尝试动作”四类信息，便于展示问题映射 Agent，而不是直接宣称根因。
- 核心题答案用于制造可解释的初步风险信号，不代表正式咨询结论。
- 如果页面出现本文未列出的映射题，按“同类问题低分、已有基础中分、确有成熟机制高分”的原则回答。

## 统一开场流程

1. 打开在线地址，点击“先了解诊断方法”。
2. 连续浏览“诊断逻辑”和案例说明页面。
3. 等待约 1 秒，点击出现的“进入企业健康初诊”。
4. 在隐私声明中缓慢向下滚动，展示以下信息：默认不提交、当前不用于模型训练、咨询授权与案例授权分离。
5. 滚动到底部，勾选/点击“我已阅读并知晓”，进入企业背景页。

## 样本一：云帆协同科技——跨部门协同与交付失速（主推荐）

### 适合展示

问题文本解析、多个经营事件合并、组织机制判断、案例与咨询师推荐。故事线最完整，建议作为主视频。

### 企业背景填写

| 字段 | 填写/选择内容 |
|---|---|
| 企业名称 | 云帆协同科技有限公司（演示） |
| 行业 | 企业服务 / SaaS |
| 规模 | 101—300人 |
| 本次由谁回答 | 业务 / 职能负责人 |

### 企业问题文本（直接粘贴）

> 从今年第二季度开始，销售、产品和实施团队在重点客户项目上频繁出现跨部门协同问题。需求变更后职责不清，方案和报价需要多轮审批并等待老板拍板，平均交付周期由45天延长到70天，返工增加，两个团队连续三个季度完不成目标。我们已经增加周会和项目群，但信息仍然重复传递，管理者经常亲自下场救火，问题没有明显改善。

### 页面操作

1. 展示 Agent 识别出的“权责与协同”“交付与流程”“人才与绩效”等待核验信号。
2. 点击“继续选择经营事件”。
3. 依次选择：
   - `跨部门事项反复卡住`
   - `交付延期或返工频发`
   - `团队连续完不成目标`
4. 将 `跨部门事项反复卡住` 设为“当前优先问题”。
5. 保留 Agent 默认加入的候选核验题，点击“进入核心判断”。

### 核心题回答口径

| 题项/关键词 | 选择答案 |
|---|---|
| Q8 组织架构匹配性 | 部分开展但不稳定 |
| Q9 权责与授权机制 | 尚未建立或基本无效 |
| Q10 流程管理规范化 | 部分开展但不稳定 |
| Q41 组织氛围与团队凝聚力 | 部分开展但不稳定 |
| Q11 人力资源规划与结构 | 基本建立并能够执行 |
| Q17 质量管理体系 | 部分开展但不稳定 |
| Q31 督办与跟踪反馈机制 | 尚未建立或基本无效 |
| Q4 战略解码与执行落地 | 部分开展但不稳定 |
| Q14 绩效管理体系 | 部分开展但不稳定 |
| Q30 战略执行分解与责任落实 | 尚未建立或基本无效 |
| Q32 执行力文化与结果导向 | 部分开展但不稳定 |
| 页面额外出现的“协同/交付/权责”题 | 部分开展但不稳定 |

### 录屏时重点展示

- “多个事件合并为核心题”，说明系统不会把三个问题简单相加。
- 结果中强调：当前是权责、流程和目标机制的风险信号，不是“员工执行力差”的既定结论。
- 打开业务路由，重点展示：
  - 组织、流程与人效业务单元
  - 精益运营/交付相关业务单元（如页面出现）
  - 相似案例卡片及“模拟已授权展示”标记
  - 咨询师 `周睿`，以及其他系统给出的候选人
- 最后打开“申请咨询”授权区，但只生成授权预览，不必提交。

### 建议旁白

> 系统没有因为企业说“执行力差”就直接推荐培训，而是把自述映射为待核验题，再结合真实发生的协同、交付和目标事件，形成组织权责与流程机制的初步方向。结果随后连接相似案例、对应业务单元和咨询师，但只有企业主动授权后，咨询公司才能看到信息。

## 样本二：精拓智能装备——制造交付、质量与数字化脱节

### 适合展示

制造业场景、精益运营和数字化业务分流，突出一个企业问题可能需要多个业务单元联合解决。

### 企业背景填写

| 字段 | 填写/选择内容 |
|---|---|
| 企业名称 | 精拓智能装备股份有限公司（演示） |
| 行业 | 制造业 |
| 规模 | 300人以上 |
| 本次由谁回答 | 业务 / 职能负责人 |

### 企业问题文本（直接粘贴）

> 近六个月订单增长，但生产交付延期和返工持续增加，准时交付率从90%下降到72%，关键工序一次合格率下降，库存和加急运输成本同时上升。公司去年上线ERP和生产报表系统，但计划、采购和车间仍大量使用手工表格，数据口径不一。我们尝试增加库存、要求每日填报和临时加班，交付短期改善后又反复。

### 页面操作

1. 观察 Agent 是否识别“交付与流程”“数字化与数据”“盈利与现金”信号。
2. 点击“继续选择经营事件”。
3. 依次选择：
   - `交付延期或返工频发`
   - `数字工具上线但没人用`
   - `价格战或毛利下降`
4. 将 `交付延期或返工频发` 设为优先问题。
5. 保留默认映射题并进入核心判断。

### 核心题回答口径

| 题项/关键词 | 选择答案 |
|---|---|
| Q10 流程管理规范化 | 部分开展但不稳定 |
| Q11 人力资源规划与结构 | 基本建立并能够执行 |
| Q17 质量管理体系 | 尚未建立或基本无效 |
| Q31 督办与跟踪反馈机制 | 部分开展但不稳定 |
| Q28 信息化系统覆盖 | 基本建立并能够执行 |
| Q29 数据驱动决策 | 尚未建立或基本无效 |
| Q52 流程创新与运营敏捷性 | 部分开展但不稳定 |
| Q56 变革管理与阻力化解 | 尚未建立或基本无效 |
| Q2 核心竞争力资源掌握 | 基本建立并能够执行 |
| Q5 商业模式清晰度 | 基本建立并能够执行 |
| Q21 产品体系与定价权 | 部分开展但不稳定 |
| Q25 财务管理与预算体系 | 部分开展但不稳定 |
| 页面额外出现的“交付/质量/数据”题 | 部分开展但不稳定 |

### 录屏时重点展示

- Q28“系统已覆盖”与 Q29“数据不能支持决策”形成有解释力的对照：问题不是有没有系统，而是流程、数据和采纳机制。
- 重点展示制造业相似案例卡片。
- 重点展示精益运营与供应链业务单元、数字化转型业务单元。
- 咨询师重点看 `许衡`、`赵清越`，说明最终名单仍需真实履历和档期数据复核。

### 建议旁白

> 这个案例展示系统如何避免把数字化问题等同于“再买一个系统”。ERP已经上线，但计划协同、质量闭环、数据口径和一线采纳仍然断开，因此路由同时连接精益运营与数字化经营能力。

## 样本三：星澜新消费集团——增长投入与新业务商业化失效

### 适合展示

CEO视角、战略与资本配置路由、创新组合与第二增长曲线判断。

### 企业背景填写

| 字段 | 填写/选择内容 |
|---|---|
| 企业名称 | 星澜新消费集团有限公司（演示） |
| 行业 | 消费品 |
| 规模 | 300人以上 |
| 本次由谁回答 | 创始人 / 总经理 |

### 企业问题文本（直接粘贴）

> 过去两年公司为寻找第二曲线持续增加增长投入，先后启动8个新产品和3个新业务试点，但大部分没有完成商业化，老业务营收增长放缓，整体利润和现金流开始承压。各事业部都认为自己的项目应该继续投入，缺少统一的阶段门、退出标准和负责人。我们已经做过年度战略规划并增加研发预算，但投资回报仍不清晰。

### 页面操作

1. 展示 Agent 识别出的“增长与市场”“创新与新业务”“盈利与现金”等信号。
2. 点击“继续选择经营事件”。
3. 依次选择：
   - `增长投入长期没有回报`
   - `创新投入无法形成新业务`
   - `商业模式不再赚钱`
4. 将 `增长投入长期没有回报` 设为优先问题。
5. 保留默认映射题并进入核心判断。

### 核心题回答口径

| 题项/关键词 | 选择答案 |
|---|---|
| Q25 财务管理与预算体系 | 部分开展但不稳定 |
| Q33 资本规划与企业融资 | 部分开展但不稳定 |
| Q44 创新战略与资源保障 | 基本建立并能够执行 |
| Q45 创新治理与项目组合 | 尚未建立或基本无效 |
| Q49 新商业模式探索与落地 | 部分开展但不稳定 |
| Q55 创新成果转化与商业化 | 尚未建立或基本无效 |
| Q5 商业模式清晰度 | 部分开展但不稳定 |
| Q21 产品体系与定价权 | 部分开展但不稳定 |
| Q50 商业模式复盘与动态优化 | 尚未建立或基本无效 |
| Q1 行业发展阶段判断 | 基本建立并能够执行 |
| Q3 战略规划清晰度 | 基本建立并能够执行 |
| Q20 市场洞察与调研 | 部分开展但不稳定 |
| 页面额外出现的“创新/增长/资本配置”题 | 部分开展但不稳定 |

### 录屏时重点展示

- Q44“有资源”与 Q45/Q55“治理和转化不足”的差异，表明系统没有把问题简化为投入不足。
- 重点展示战略与集团管控、资本运作与融资、品牌营销与增长等候选业务单元。
- 重点展示创新商业化相似案例。
- 咨询师重点看 `林知远`、`唐嘉禾`；正式匹配仍需真实项目经验和利益冲突核验。

### 建议旁白

> CEO看到的表面问题是新业务没有回报，但系统进一步区分了战略选择、资本配置、创新组合治理和商业化转化。这里的路由不是按某一道问卷题机械分流，而是把企业自述、经营事件与核心判断合并后，再连接可能需要的咨询能力。

## 推荐的视频结构（3—5分钟）

1. **0:00—0:25**：首页定位——“从经营事件开始，而不是直接做一份泛问卷”。
2. **0:25—0:50**：诊断方法与隐私说明。
3. **0:50—1:30**：填写样本一背景和问题，展示 Agent 实时映射。
4. **1:30—2:10**：选择三个经营事件并指定优先问题。
5. **2:10—2:45**：回答核心题，强调重复题只回答一次。
6. **2:45—3:35**：展示风险信号、矛盾/待核验关系和下一步证据要求。
7. **3:35—4:20**：展示业务单元、已授权模拟案例和咨询师匹配。
8. **4:20—4:45**：打开授权预览，说明企业主动授权后才建立咨询连接，案例展示需另行授权。

## 录屏避坑

- 不要把所有题都选择最低档，否则画面像为了制造问题而操纵结果。
- 不要说“AI已经确认根因”；应说“Agent形成待核验信号和问题机制假设”。
- 不要把模拟案例指标说成和君真实项目成绩。
- 不要点击刷新或关闭页面，当前原型会清除本次填写内容。
- 建议提前准备文本复制粘贴，并用样本一完整录制；样本二、三可分别剪成30—60秒的行业扩展示例。
