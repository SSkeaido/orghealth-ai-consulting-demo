## V3.1 内容证据版

访问 `?v=3.1`。该版本保留 V3 的经营事件路由，并在工作台的“补充证据”中为 Q2、Q5、Q6、Q9、Q10、Q29、Q41、Q56 增加结构化内容采集。完成标记只表示达到最低分析信息量，不代表已经确认问题或根因。

一个以证据为中心的企业健康度诊断个人项目。它把传统的员工问卷改造成“企业资料 + 访谈事实 + 顾问核验 + 可追溯评分”的工作流。

## 当前原型

- 极简项目创建页：企业背景与资料上传
- 证据资料库：文件状态、核验状态、诊断项关联
- 18 模块诊断工作区：筛选、评分、状态更新和证据结论
- 项目概览：综合健康度、进度、风险和五维雷达图
- 报告预览：核心判断、证据覆盖和改善重点
- 响应式桌面/移动端布局

当前版本是前端交互原型，使用演示数据；上传文件不会发送到服务器。

## 运行

```bash
pnpm install
pnpm dev
```

本地同时启动网页与硅基流动代理：

```bash
pnpm dev:local
```

启动后访问 `http://localhost:5173/?v=3.1`。本地模型代理监听 `127.0.0.1:8787`，只接受当前电脑的连接。

### 硅基流动语义解析（可选）

前端不会保存或携带 API Key。模型调用必须经过同源服务端接口：

1. 将项目部署到支持 `/api` 函数的平台（仓库包含 Vercel API 路由）。
2. 在服务端配置 `SILICONFLOW_API_KEY`。
3. 可用 `SILICONFLOW_MODEL` 切换模型，默认 `deepseek-ai/DeepSeek-V3.2`。
4. 构建前设置 `VITE_SEMANTIC_API_URL=/api/analyze`。

未配置接口时，产品自动使用原有本地规则映射。不要把硅基流动 API Key 写进 `VITE_*` 变量或前端源码。

质量检查：

```bash
pnpm lint
pnpm build
```
